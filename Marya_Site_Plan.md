## 1. ВВЕДЕНИЕ

### Концепция и миссия

**"Марья"** — это не просто магазин косметики. Это персональный советник по уходу за кожей, который говорит на вашем языке: без латинских терминов, без страха ошибиться, без перегруза. Миссия бренда: **"Простой уход, который действительно работает"**.

В мире, где полки ломятся от сывороток с «революционными» формулами, "Марья" занимает позицию честного проводника: объясняет составы, подбирает рутину под конкретную кожу и помогает превратить уход в устойчивую привычку — не подвиг раз в месяц, а 5 минут утром и вечером.

### Ключевые фичи платформы

- **AI Skin-Coach** — квиз из 7 вопросов (тип кожи, проблемы, образ жизни) + опциональный camera-скан с явным согласием → персональная рутина с объяснением "почему именно это".
- **Routine Builder** — интерактивный конструктор утро/вечер с проверкой совместимости активных ингредиентов в реальном времени и визуальным расписанием нанесения.
- **SkinCare Duo** — режим совместного выбора: покупатель + подруга/консультант в одной сессии. AI-модератор предупреждает о конфликтах активов и предлагает компромиссы.
- **Геймификация привычек** — streak-счётчик, челленджи "7 дней сияния", бейджи, мягкие nudge-напоминания через push/email.
- **Web3-лояльность (опционально)** — NFT-паспорт продукта: подлинность, история состава, эксклюзивные перки для держателей.

### Цели и метрики

| Метрика | Цель |
|---|---|
| Конверсия (CVR) | 12–18% |
| Bounce rate | < 30% |
| Session duration | > 4–6 мин |
| Repeat purchase rate | +40–60% |
| LTV | $150–250 |

Модель монетизации: **D2C + подписка на рутину/рефилы + бандлы**. Акцент на zero-party data — пользователь сам делится предпочтениями в обмен на точные рекомендации, без скрытого трекинга.

### 2.3 СТРАНИЦА ПРОДУКТА (PDP)

**Цель:** Снять все страхи ("подойдёт ли?", "как использовать?", "с чем сочетать?") и конвертировать.

#### Ключевые блоки

**[ГАЛЕРЕЯ — 60% экрана desktop]**
- 5–7 фото: продукт, текстура, нанесение, before-context (кожа без претензий к результату)
- Кнопка `[AR-предпросмотр]` → WebXR-наложение текстуры крема (Three.js, с fallback на видео)
- Zoom on hover (desktop) / pinch (mobile)

**[ПРАВАЯ КОЛОНКА]**
- Название / краткий дескриптор ("Увлажняющий крем с гиалуронкой для сухой кожи")
- Рейтинг: ★★★★☆ 4.8 (1 240 отзывов) → anchor к отзывам
- Ключевые активы: 3 пилюли ("Гиалуронка 2%" / "Пантенол" / "Без отдушек")
- **"Подойдёт вам?"** → мини-чекбокс по профилю (зелёный ✓ / жёлтый ? / `[Пройти квиз]`)
- Объём/вариант: toggle-кнопки (30мл / 50мл)
- Цена: обычная + цена по подписке со значком "−15% каждый месяц"
- CTA stack:
  - `[Добавить в рутину]` (primary, полная ширина)
  - `[Купить сейчас]` (secondary)
  - `[Добавить в набор с...]` → cross-sell

**[ВКЛАДКИ КОНТЕНТА]**
- **Состав:** полный INCI-список с расшифровкой топ-5 ингредиентов простым языком
- **Кому подходит / Кому не рекомендуем:** чёткий список без "посоветуйтесь с врачом" (только если реально нужно)
- **Как использовать:** пошагово (1–2–3), иконки, "утром или вечером?", количество
- **Совместимость:** "Хорошо сочетается с: ниацинамид, пептиды" / "Не используйте вместе с: AHA/BHA, ретинол"
- **FAQ:** 5–7 вопросов (schema.org FAQPage), отвечаем коротко

**[ОТЗЫВЫ]**
- Сортировка: Полезные / Новые / С фото / По типу кожи
- Фильтр по типу кожи — ключевой дифференциатор
- UGC-фото (с согласия)
- Форма отзыва: тип кожи + проблема + звёзды + текст + фото

**[CROSS-SELL "СОБЕРИ РУТИНУ"]**
- "С этим продуктом берут:" → 3 карточки с логикой этапа рутины
- "Добавить все 3 со скидкой 15%" → `[Собрать набор]`

### 2.4 ПЕРСОНАЛИЗАЦИЯ (SKIN-QUIZ + ROUTINE BUILDER)

**Цель:** Собрать zero-party data, устранить страх выбора, сформировать персональную рутину.

#### Skin-Quiz (7 шагов)

**Шаг 1 — Тип кожи**
- "Как ведёт себя кожа к середине дня?" → Сухая / Жирная / Комбинированная / Нормальная / Не знаю
- Micro-copy: "Это поможет подобрать правильную текстуру"

**Шаг 2 — Главная проблема (multiselect, max 3)**
- Акне / Пигментация / Морщины / Тусклость / Сухость / Чувствительность / Всё ок

**Шаг 3 — Образ жизни**
- "Сколько шагов в уходе вы готовы делать?" → 2–3 / 4–5 / 6+ / Зависит от настроения

**Шаг 4 — Ограничения**
- Мультиселект: Беременность / Кормление / Аллергия на...  / Нет ограничений
- Примечание: "Это помогает исключить потенциально чувствительные ингредиенты. Мы не ставим диагнозов."

**Шаг 5 — Бюджет**
- "Готовы потратить на рутину в месяц" → до 1 500₽ / 1 500–3 000₽ / 3 000₽+

**Шаг 6 (опциональный) — Camera-scan**
- "Хотите, чтобы AI увидел кожу и дал пояснение? (Без сохранения, только анализ)"
- Явный чекбокс согласия + иконка замка "Фото не сохраняется"
- Fallback: "Пропустить → продолжить без сканирования"
- После скана: "Мы видим признаки обезвоженности в Т-зоне — рекомендуем гиалуронку." Без диагнозов, без страшных слов.

**Шаг 7 — Email для сохранения рутины**
- "Сохранить рутину на почту" → необязательно (гостевой режим сохраняет в localStorage)

**Результат квиза:**
- Персональная страница рутины: Утро (3 шага) / Вечер (3 шага)
- Каждый шаг: продукт + "почему именно он" (1–2 предложения)
- CTA: `[Добавить все в корзину]` / `[Настроить рутину]`
- Предупреждения совместимости: если выбраны конфликтующие продукты — жёлтый баннер "Совет: не используйте ретинол и AHA в одну ночь. Чередуйте."

#### Routine Builder (drag-and-drop)

- 2 колонки: Утро / Вечер
- Этапы зафиксированы: Очищение → Тоник → Сыворотка → Увлажнение → SPF (только утро) / Ночной крем (только вечер) → Маска (опционально)
- Пользователь перетаскивает продукты из "Моей полки" или ищет из каталога
- В реальном времени: "⚠️ Ниацинамид + Витамин С — лучше в разное время суток"
- Кнопка `[Сохранить рутину]` → дашборд + напоминания

### 2.7 CHECKOUT + ЛИЧНЫЙ КАБИНЕТ (DASHBOARD)

- Оплата: Stripe / Apple Pay / Google Pay / СБП (для РФ)

#### Личный кабинет / Dashboard

**[МОЯ РУТИНА]**
- Визуальная timeline: утро 7:00 → вечер 22:00

**[ТРЕКЕР STREAK]**
- Календарь: зелёные дни (выполнено), серые (пропущено)

**[МОИ БЕЙДЖИ И ПРОГРЕСС]**
- Прогресс-бар уровня в программе лояльности

**[ИСТОРИЯ ЗАКАЗОВ]**
- Статус / Трек / `[Повторить заказ]` / `[Оставить отзыв]`

**[НАСТРОЙКИ ПРОФИЛЯ]**
- Тип кожи / проблемы / ограничения (обновить в любой момент)
- Управление подписками / напоминаниями
- Privacy: `[Скачать мои данные]` / `[Удалить аккаунт и данные]` (GDPR)

```javascript
  try {
    const stream = await navigator.mediaDevices.getUserMedia({
      video: { facingMode: 'user', width: { ideal: 640 }, height: { ideal: 480 } }
    });
    const video = document.getElementById('camera-preview');
    video.srcObject = stream;
    document.getElementById('scan-consent').hidden = true;

    // Захват кадра через 3 сек для анализа
    setTimeout(() => {
      const canvas = document.getElementById('scan-canvas');
      canvas.width = video.videoWidth;
      canvas.height = video.videoHeight;
      canvas.getContext('2d').drawImage(video, 0, 0);
      const base64 = canvas.toDataURL('image/jpeg', 0.7);

      // Отправка на edge function (OpenAI Vision API)
      sendForAnalysis(base64);

      // Остановить камеру
      stream.getTracks().forEach(t => t.stop());
    }, 3000);
  } catch (err) {
    // Fallback: предложить продолжить без скана
    showFallback();
  }
});

  const res = await fetch('/api/skin-scan', {

  });
```

```javascript
// ar-texture-preview.js
async function startARPreview(productTextureUrl) {
  if (!navigator.xr) {
    // Fallback: видео-демонстрация текстуры
    showVideoFallback(productTextureUrl);
    return;
  }

  const supported = await navigator.xr.isSessionSupported('immersive-ar');
  if (!supported) { showVideoFallback(productTextureUrl); return; }

  const session = await navigator.xr.requestSession('immersive-ar', {
    requiredFeatures: ['hit-test']
  });

  // Инициализация Three.js renderer для XR
  const renderer = new THREE.WebGLRenderer({ antialias: true, alpha: true });
  renderer.xr.enabled = true;
  renderer.xr.setSession(session);

  // Текстура продукта
  const texture = new THREE.TextureLoader().load(productTextureUrl);
  const plane = new THREE.Mesh(
    new THREE.PlaneGeometry(0.1, 0.1),
    new THREE.MeshBasicMaterial({ map: texture, transparent: true })
  );

  const scene = new THREE.Scene();
  scene.add(plane);

  renderer.setAnimationLoop((_, frame) => {
    if (frame) renderer.render(scene, camera);
  });
}
```

### 6.1 Skin Photo Diary — прогресс-дневник

**Референс:** Curology, Proven Skincare

**Концепция:** Пользователь делает фото кожи раз в неделю прямо в дашборде. Фото хранятся только на устройстве (on-device, не на серверах). Через 4 недели — сравнение через flip-карточку "было/стало". Без медицинских обещаний — только субъективная динамика.

**Ключевые блоки:**
- Кнопка `[Сфотографировать сегодня]` в трекере → открывает камеру с рамкой-гайдом (свет, расстояние)
- Галерея недель: горизонтальный scroll, каждое фото подписано датой и этапом рутины
- Flip-сравнение: слайдер "неделя 1 / неделя 4"
- Шеринг (опционально): `[Поделиться историей в сообществе]` — с явным согласием, с возможностью заблюрить

**UX-детали:**
- Явное согласие при первом открытии: "Фото хранятся только на вашем устройстве"
- `[Удалить все фото]` — одна кнопка в настройках профиля
- Без геймификации на основе "улучшений" — только нейтральная хронология

**Влияние на метрики:** снижение возвратов, рост retention (пользователь сам видит динамику и не бросает рутину).

### 6.2 Гибкий менеджмент подписки

**Референс:** Function of Beauty, Dollar Shave Club, Recharge

**Концепция:** Подписка на рутину/рефилы должна быть лёгкой в управлении — иначе пользователь её отменит вместо паузы.

**Возможности в личном кабинете:**

| Действие | Описание |
|---|---|
| Пауза | На 1, 2 или 3 месяца — без потери скидки |
| Пропустить месяц | Разовый skip следующей поставки |
| Swap продукта | Поменять один продукт в подписке без пересоздания |
| Изменить дату | Сдвинуть следующую доставку на нужную дату |
| Отменить | Всегда доступна, без скрытых шагов (по GDPR-духу) |

**Умный прогноз остатка:**
- На карточке подписного продукта: "Хватит примерно на 18 дней" (расчёт по объёму + частоте использования из трекера)
- За 5 дней до окончания: пуш/email "Ваш крем заканчивается — следующая поставка 15 марта"

**Microcopy при отмене:** "Жаль, что уходите. Хотите поставить на паузу вместо отмены?" → `[Пауза на месяц]` `[Всё равно отменить]`

### 6.4 Ingredient Conflict Tool — публичный инструмент

**Референс:** Paula's Choice ("Ingredient Dictionary"), INCI Decoder

**Концепция:** Отдельная SEO-страница `/ingredients/check` — без авторизации, открытый инструмент. Привлекает органический трафик по запросам "ретинол и кислоты вместе", "можно ли смешивать ниацинамид и витамин С".

**Интерфейс:**
```
[Ингредиент 1: ___________▼]  +  [Ингредиент 2: ___________▼]
                    [Проверить совместимость]

⚠️ Ретинол + AHA-кислоты
Лучше чередовать: ретинол — пн/ср/пт, AHA — вт/чт.
Одновременное применение может усилить раздражение.

[Посмотреть продукты с ретинолом →]  [Посмотреть продукты с AHA →]
```

**Дополнительно:**
- Глоссарий: страница каждого ингредиента (schema.org Article) с карточкой "с чем сочетать / не сочетать"

**SEO-ценность:** The Ordinary построила cult-following именно на такой открытой образовательной базе. Низкая стоимость реализации, высокий органический ROI.

### 6.5 "Founding Member" / Product Drop механика

**Референс:** Glossier (фавориты), Drunk Elephant (лимитки), e.l.f. Cosmetics

- Кнопка `[♡ Сохранить]` на карточке товара / PDP

- Лимит: "Осталось 47 штук" — реальный счётчик
- После распродажи: `[Уведомить о следующем дропе]`

### 6.7 Seasonal Skin Recalibration

**Референс:** Clinique ("Skin Type Re-evaluation"), Kiehl's

**Концепция:** Автоматический триггер 4 раза в год (март, июнь, сентябрь, декабрь) для пользователей с сохранённой рутиной.

**Email/пуш:**
> "Пришла весна — кожа меняется вместе с погодой. Обновить рутину под новый сезон?"

→ Ведёт на укороченный квиз (3 вопроса, не 7 — данные уже есть) с сезонным пресетом.

**Сезонные пресеты:**
- Зима: акцент на питание, защита от ветра, меньше кислот
- Весна: детокс, выравнивание тона, первый SPF-напоминатель
- Лето: лёгкие текстуры, усиление SPF, контроль блеска
- Осень: восстановление, увлажнение, введение ретинола

**В каталоге:** сезонный баннер "Актуально сейчас для вашей кожи" → отфильтрованная подборка.

### 6.8 Live Video Consultation

**Референс:** Dermalogica ("Face Mapping"), Charlotte Tilbury (virtual beauty advisor)

**Концепция:** Слот 15–20 минут с косметологом-консультантом.

**Бронирование:**

- Выбор формата: видео (Zoom/WebRTC) / текстовый чат
- Доступность: бесплатно при заказе от 3 000₽, иначе 500₽ (засчитываются в следующий заказ)

2. Консультация 15 мин
3. После: рекомендации сохраняются в профиль автоматически
4. Одним кликом: `[Добавить рекомендованное в рутину]`

**Метрика:** конверсия после live-консультации составляет 60–80% vs 3–5% холодного трафика (данные Dermalogica).

### 6.9 "Proven Ingredients" Transparency Badges

**Референс:** The Ordinary, Drunk Elephant, Paula's Choice

**На странице продукта (PDP):**
Под описанием каждого ключевого активного ингредиента — раскрывающаяся плашка:

```
[Ниацинамид 5%]  ▼
──────────────────────────────────────────
Что делает: сужает поры, выравнивает тон, снижает выработку кожного сала.
Концентрация: 5% — клинически эффективная доза.
Исследование: Kim et al., 2022 — "Topical niacinamide reduces sebum production" [PubMed →]
Без: искусственных красителей, сульфатов.
```

**Принципы:**
- Только ссылки на реальные peer-reviewed источники (PubMed, NCBI)
- Формулировки без клинических обещаний: "исследования показывают", "может способствовать"
- Если исследований нет — не добавлять значок

**Бейдж на карточке товара:** "Клинически изученные активы" — небольшая иконка колбы, при hover → список активов с пруфами.

**SEO + доверие:** The Ordinary построила $460M revenue именно на открытой ingredient transparency.
