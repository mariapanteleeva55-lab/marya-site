## 1. ВВЕДЕНИЕ

### Концепция и миссия

**"Марья"** — это не просто магазин косметики. Это персональный советник по уходу за кожей, который говорит на вашем языке: без латинских терминов, без страха ошибиться, без перегруза. Миссия бренда: **"Простой уход, который действительно работает"**.

В мире, где полки ломятся от сывороток с «революционными» формулами, "Марья" занимает позицию честного проводника: объясняет составы, подбирает рутину под конкретную кожу и помогает превратить уход в устойчивую привычку — не подвиг раз в месяц, а 5 минут утром и вечером.

### Ключевые фичи платформы

- **AI Skin-Coach** — квиз из 7 вопросов (тип кожи, проблемы, образ жизни) + опциональный camera-скан с явным согласием → персональная рутина с объяснением "почему именно это".
- **Routine Builder** — интерактивный конструктор утро/вечер с проверкой совместимости активных ингредиентов в реальном времени и визуальным расписанием нанесения.
- **SkinCare Duo** — режим совместного выбора: покупатель + подруга/консультант в одной сессии. AI-модератор предупреждает о конфликтах активов и предлагает компромиссы.
- **Геймификация привычек** — streak-счётчик, челленджи "7 дней сияния", бейджи, мягкие nudge-напоминания через Telegram / WhatsApp / ВКонтакте / Push — на выбор пользователя.
- **Web3-лояльность (опционально)** — NFT-паспорт продукта: подлинность, история состава, эксклюзивные перки для держателей.

### Цели и метрики

| Метрика | Цель |
|---|---|
| Конверсия (CVR) | 12–18% |
| Bounce rate | < 30% |
| Session duration | > 4–6 мин |
| Repeat purchase rate | +40–60% |
| LTV | $150–250 |

Модель монетизации: **D2C + подписка на рутину/рефилы + бандлы**. Акцент на zero-party data — пользователь сам делится предпочтениями в обмен на точные рекомендации, без скрытого трекинга.

- Ключевые активы: 3 пилюли ("Гиалуронка 2%" / "Пантенол" / "Без отдушек")

- **Как использовать:** пошагово (1–2–3), иконки, "утром или вечером?", количество

**[ОТЗЫВЫ]**

### 2.4 ПЕРСОНАЛИЗАЦИЯ (SKIN-QUIZ + ROUTINE BUILDER)

**Цель:** Собрать zero-party data, устранить страх выбора, сформировать персональную рутину.

#### Skin-Quiz (7 шагов)

**Шаг 1 — Тип кожи**
- "Как ведёт себя кожа к середине дня?" → Сухая / Жирная / Комбинированная / Нормальная / Не знаю
- Micro-copy: "Это поможет подобрать правильную текстуру"

**Шаг 2 — Главная проблема (multiselect, max 3)**
- Акне / Пигментация / Морщины / Тусклость / Сухость / Чувствительность / Всё ок

**Шаг 3 — Образ жизни**
- "Сколько шагов в уходе вы готовы делать?" → 2–3 / 4–5 / 6+ / Зависит от настроения

**Шаг 4 — Ограничения**
- Мультиселект: Беременность / Кормление / Аллергия на...  / Нет ограничений
- Примечание: "Это помогает исключить потенциально чувствительные ингредиенты. Мы не ставим диагнозов."

**Шаг 5 — Бюджет**
- "Готовы потратить на рутину в месяц" → до 1 500₽ / 1 500–3 000₽ / 3 000₽+

**Шаг 6 (опциональный) — Camera-scan**
- "Хотите, чтобы AI посмотрел на кожу и дал подсказку?"
- Явный чекбокс согласия с честным текстом: "Фото отправляется на защищённый сервер для анализа и удаляется сразу после ответа. Мы не храним изображения." + иконка замка
- Fallback: "Пропустить → продолжить без сканирования"
- После скана: "Мы замечаем признаки обезвоженности в области лба — рекомендуем гиалуронку." Без диагнозов, без страшных слов.
- ⚠️ **Технически:** фото уходит на edge-функцию → OpenAI Vision API → ответ → фото нигде не сохраняется. Это должно быть отражено в политике конфиденциальности.

**Шаг 7 — Контакт для сохранения рутины**
- "Куда отправить рутину?" → выбор: Telegram / WhatsApp / Email → необязательно (гостевой режим сохраняет в localStorage)

**Результат квиза:**
- Персональная страница рутины: Утро (3 шага) / Вечер (3 шага)
- Каждый шаг: продукт + "почему именно он" (1–2 предложения)
- CTA: `[Добавить все в корзину]` / `[Настроить рутину]`
- Предупреждения совместимости: если выбраны конфликтующие продукты — жёлтый баннер "Совет: не используйте ретинол и AHA в одну ночь. Чередуйте."

#### Routine Builder (drag-and-drop)

- 2 колонки: Утро / Вечер
- Этапы зафиксированы: Очищение → Тоник → Сыворотка → Увлажнение → SPF (только утро) / Ночной крем (только вечер) → Маска (опционально)
- Пользователь перетаскивает продукты из "Моей полки" или ищет из каталога
- Мобайл: drag-and-drop заменён на кнопки "↑ ↓" и "+" — стандартный touch drag ненадёжен на iOS Safari
- **Лимит:** максимум 7 продуктов в каждой сессии (утро / вечер). При попытке добавить 8-й: "Рутина уже полная — уберите что-то, чтобы добавить новое"
- В реальном времени: "⚠️ Ниацинамид + Витамин С — лучше в разное время суток"
- **Если товар закончился на складе:** отображается с меткой "Нет в наличии" и кнопкой `[Уведомить о поступлении]` — не удаляется из рутины
- Кнопка `[Сохранить рутину]` → дашборд + напоминания

### 2.6 СООБЩЕСТВО

**Цель:** Социальное доказательство + retention через принадлежность.

#### Блоки

**[UGC-ИСТОРИИ]**
- "Рутины пользователей" — карточки: фото (опционально) + тип кожи + проблема + используемые продукты + "3 месяца спустя: ощущения"
- Без клинических обещаний. Только субъективный опыт.
- `[Поделиться своей историей]` → форма с согласием на публикацию

**[SKINCARE DUO — совместный выбор]**
- Кнопка `[Выбирать с подругой]` → генерация ссылки-токена (24ч жизни, одноразовая)
- **Протокол синхронизации:** WebSocket-соединение (socket.io) — оба клиента подключены к одной комнате по room-id из токена
- Real-time: оба участника видят одни и те же продукты и действия друг друга
- AI-модератор слева: "У вас разные типы кожи — вот что подойдёт обеим" / "⚠️ Это средство может раздражать чувствительную кожу"
- **Edge-cases:**
  - Один участник отключился → показываем "Подруга временно недоступна. Ждём..." (30 сек), затем режим ожидания
  - Истекли 24ч → страница с сообщением "Сессия завершена — создайте новую" + кнопка
  - Оба добавляют товар одновременно → merge без конфликта, оба видят обновление
- Итог: общий набор + каждый добавляет к себе в корзину независимо

**[КОНСУЛЬТАЦИИ / ВЕБИНАРЫ]**
- Расписание: live-вебинары с косметологами
- Запись: архив прошлых
- "Задать вопрос" → асинхронный AI-коуч (отвечает на основе базы знаний, не ставит диагнозы)

**[WEB3-ЛОЯЛЬНОСТЬ (ОПЦИОНАЛЬНО)]**
- NFT-паспорт продукта: QR на упаковке → история состава + сертификаты + перки держателя
- "Ранний доступ к новинкам" / "−20% для NFT-холдеров"
- Важно: необязательный слой. Работает как "премиум-клуб" без принуждения

### 2.7 CHECKOUT + ЛИЧНЫЙ КАБИНЕТ (DASHBOARD)

**Цель:** Убрать трение на пути к оплате + создать пространство для привычки и возврата.

#### Checkout

- **One-page checkout**: контакт → доставка → оплата
- Оплата: Тинькофф Касса (карты Visa/MC/МИР, СБП, SberPay, Apple Pay, Google Pay — всё через один шлюз)
- "Оформить подписку на эту рутину" → toggle с объяснением экономии: "−15% каждый месяц, отмена в 1 клик"
- Upsell: "К вашему заказу часто добавляют:" → 1 релевантный продукт, не 5
- Order confirmation: сообщение в выбранный мессенджер + `[Добавить в календарь]` (напоминание начать рутину)

#### Личный кабинет / Dashboard

**[МОЯ РУТИНА]**
- Визуальная timeline: утро 7:00 → вечер 22:00
- Статус каждого продукта: "Заканчивается через ~12 дней" → `[Заказать снова]`
- Кнопка `[Изменить рутину]` → Routine Builder

**[ТРЕКЕР STREAK]**
- Календарь: зелёные дни (выполнено), серые (пропущено)

```javascript
// camera-scan.js
document.getElementById('scan-allow').addEventListener('click', async () => {
  let stream;
  try {
    stream = await navigator.mediaDevices.getUserMedia({
      video: { facingMode: 'user', width: { ideal: 640 }, height: { ideal: 480 } }
    });

    const video = document.getElementById('camera-preview');
    video.srcObject = stream;
    document.getElementById('scan-consent').hidden = true;
    document.getElementById('scan-status').textContent = 'Подготовка камеры...';

    // Ждём canplay — гарантия что videoWidth/videoHeight не равны 0
    await new Promise((resolve) => {
      video.addEventListener('canplay', resolve, { once: true });
    });

    document.getElementById('scan-status').textContent = 'Анализируем...';

    const canvas = document.getElementById('scan-canvas');
    canvas.width = video.videoWidth;
    canvas.height = video.videoHeight;
    const ctx = canvas.getContext('2d');
    if (!ctx) throw new Error('Canvas context unavailable');
    ctx.drawImage(video, 0, 0);
    const base64 = canvas.toDataURL('image/jpeg', 0.7);

    // Останавливаем камеру до отправки — пользователь видит что камера выключена
    stream.getTracks().forEach(t => t.stop());
    video.srcObject = null;

    await sendForAnalysis(base64);

  } catch (err) {
    // Останавливаем камеру при любой ошибке
    if (stream) stream.getTracks().forEach(t => t.stop());
    console.error('[camera-scan]', err);
    showFallback();
  }
});

async function sendForAnalysis(imageBase64) {
  const controller = new AbortController();
  const timeout = setTimeout(() => controller.abort(), 15000); // 15 сек таймаут

  try {
    const res = await fetch('/api/skin-scan', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ image: imageBase64 }),
      signal: controller.signal,
    });

    if (!res.ok) throw new Error(`Server error: ${res.status}`);

    const data = await res.json();
    if (!data.hints) throw new Error('Invalid response format');

    displayScanHints(data.hints); // Показываем бережные подсказки, не диагнозы

  } catch (err) {
    if (err.name === 'AbortError') {
      showFallback('Анализ занял слишком долго — продолжим без скана');
    } else {
      showFallback();
    }
    console.error('[skin-scan API]', err);
  } finally {
    clearTimeout(timeout);
    document.getElementById('scan-status').textContent = '';
  }
}
```

### WebXR AR-overlay (базовый пример)

```javascript
// ar-texture-preview.js
// ⚠️ Загружать через dynamic import — НЕ включать в основной бандл:
// const { startARPreview } = await import('./ar-texture-preview.js')

async function startARPreview(productTextureUrl) {
  // Проверка поддержки — Safari не поддерживает WebXR
  if (!navigator.xr) {
    showVideoFallback(productTextureUrl);
    return;
  }

  const supported = await navigator.xr.isSessionSupported('immersive-ar');
  if (!supported) {
    showVideoFallback(productTextureUrl);
    return;
  }

  // Динамически импортируем Three.js только здесь
  const THREE = await import('three');

  let session;
  try {
    session = await navigator.xr.requestSession('immersive-ar', {
      requiredFeatures: ['hit-test'],
    });
  } catch (err) {
    // Пользователь отказал в доступе к камере для AR
    showVideoFallback(productTextureUrl);
    return;
  }

  // Инициализация Three.js renderer
  const renderer = new THREE.WebGLRenderer({ antialias: true, alpha: true });
  renderer.setSize(window.innerWidth, window.innerHeight);
  renderer.xr.enabled = true;

  // Добавляем canvas в DOM
  const arContainer = document.getElementById('ar-container');
  arContainer.appendChild(renderer.domElement);
  arContainer.hidden = false;

  await renderer.xr.setSession(session);

  // Камера управляется XR-системой автоматически
  const camera = new THREE.PerspectiveCamera(70, window.innerWidth / window.innerHeight, 0.01, 20);
  const scene = new THREE.Scene();

  // Текстура продукта — ждём загрузки перед рендером
  const texture = await new Promise((resolve, reject) => {
    new THREE.TextureLoader().load(productTextureUrl, resolve, undefined, reject);
  });

  const plane = new THREE.Mesh(
    new THREE.PlaneGeometry(0.1, 0.1),
    new THREE.MeshBasicMaterial({ map: texture, transparent: true, side: THREE.DoubleSide })
  );
  // Размещаем плоскость перед камерой на старте
  plane.position.set(0, 0, -0.3);
  scene.add(plane);

  renderer.setAnimationLoop((_, frame) => {
    if (frame) {
      renderer.render(scene, camera);
    }
  });

  // Кнопка закрытия AR
  document.getElementById('ar-close-btn').addEventListener('click', () => {
    renderer.setAnimationLoop(null);
    session.end();
    arContainer.hidden = true;
    renderer.domElement.remove();
    renderer.dispose();
  }, { once: true });

  // Автозакрытие при завершении сессии устройством
  session.addEventListener('end', () => {
    renderer.setAnimationLoop(null);
    arContainer.hidden = true;
    renderer.domElement.remove();
    renderer.dispose();
  });
}

function showVideoFallback(productTextureUrl) {

  const fallback = document.getElementById('ar-video-fallback');
  if (fallback) {
    fallback.src = productTextureUrl.replace('.jpg', '-demo.mp4');
    fallback.hidden = false;
  }
}
```

### 6.1 Skin Photo Diary — прогресс-дневник

**Ключевые блоки:**
- Кнопка `[Сфотографировать сегодня]` в трекере → открывает камеру с рамкой-гайдом (свет, расстояние)
- Галерея недель: горизонтальный scroll, каждое фото подписано датой и этапом рутины
- Flip-сравнение: слайдер "неделя 1 / неделя 4"
- Шеринг (опционально): `[Поделиться историей в сообществе]` — с явным согласием, с возможностью заблюрить

**UX-детали:**
- Явное согласие при первом открытии: "Фото хранятся только на вашем устройстве"
- `[Удалить все фото]` — одна кнопка в настройках профиля
- Без геймификации на основе "улучшений" — только нейтральная хронология

**Влияние на метрики:** снижение возвратов, рост retention (пользователь сам видит динамику и не бросает рутину).

### 6.2 Гибкий менеджмент подписки

**Референс:** Function of Beauty, Dollar Shave Club, Recharge

**Концепция:** Подписка на рутину/рефилы должна быть лёгкой в управлении — иначе пользователь её отменит вместо паузы.

**Возможности в личном кабинете:**

| Действие | Описание |
|---|---|
| Пауза | На 1, 2 или 3 месяца — без потери скидки |
| Пропустить месяц | Разовый skip следующей поставки |
| Swap продукта | Поменять один продукт в подписке без пересоздания |
| Изменить дату | Сдвинуть следующую доставку на нужную дату |
| Отменить | Всегда доступна, без скрытых шагов (по GDPR-духу) |

**Умный прогноз остатка:**
- На карточке подписного продукта: "Хватит примерно на 18 дней" (расчёт по объёму + частоте использования из трекера)

### 6.7 Seasonal Skin Recalibration

**Референс:** Clinique ("Skin Type Re-evaluation"), Kiehl's

- Лето: лёгкие текстуры, усиление SPF, контроль блеска
- Осень: восстановление, увлажнение, введение ретинола

**В каталоге:** сезонный баннер "Актуально сейчас для вашей кожи" → отфильтрованная подборка.

### 6.9 "Proven Ingredients" Transparency Badges

**Референс:** The Ordinary, Drunk Elephant, Paula's Choice

**На странице продукта (PDP):**
Под описанием каждого ключевого активного ингредиента — раскрывающаяся плашка:

```
[Ниацинамид 5%]  ▼
──────────────────────────────────────────
Что делает: сужает поры, выравнивает тон, снижает выработку кожного сала.
Концентрация: 5% — клинически эффективная доза.
Исследование: Kim et al., 2022 — "Topical niacinamide reduces sebum production" [PubMed →]
Без: искусственных красителей, сульфатов.
```

**Принципы:**
- Только ссылки на реальные peer-reviewed источники (PubMed, NCBI)
- Формулировки без клинических обещаний: "исследования показывают", "может способствовать"
- Если исследований нет — не добавлять значок

**Бейдж на карточке товара:** "Клинически изученные активы" — небольшая иконка колбы, при hover → список активов с пруфами.

**SEO + доверие:** The Ordinary построила $460M revenue именно на открытой ingredient transparency.
